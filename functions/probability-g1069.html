AI 在機率製程中負責的 四項工作

📋

規格轉譯

從 Excel 規格書抽出 base game、FG、BuyBonus、JP、scatter、special reel、collect gate 等設計重點。
AI 自動抽取重點並列出規格矛盾與待確認項，省去人工逐頁翻讀。
結果：矛盾項即時浮現，reviewer 可直接針對問題拍板。
🏗️

架構設計

參考福星5既有 PAR sheet 與遊戲框架，拆 RTP contribution、feature gate、盤面狀態、參數表。
AI 直接對齊 Gold1–5 / ALLWAYS / SC 框架，補上福星6差異：MG collect gate、FG special reel、JP gate。
結果：架構草案完整覆蓋福星6新增設計，無漏項。
⚙️

程式生成

從架構草案生成 Go quicktest scaffold，odds.go 維持純參數檔，flow 邏輯放 game_process.go。
AI 生成 v1 scaffold，依 reviewer 回饋修正 v2：移除 odds.go 的 executable content，補 FG 盤面狀態邏輯。
結果：v2 go test ./... ＆ go build ./... 全部 PASS。
✅

驗證協助

scaffold 產出後需要立即確認可 build、可 test，正式 PAR 值補完後還要跑 Monte Carlo。
AI 自動執行 go test / go build，生成後即驗證；正式值到位後可接 RTP / SD / CI / hit rate 輸出。
結果：第一版可 build 可 review，降低 template drift 風險。
✅ 本次完成：機率架構草案 · v1 scaffold · v2 scaffold · go test PASS · go build PASS 🔒 範圍定位：工程骨架，非正式 RTP 版；PAR 值補完後才跑正式驗證
