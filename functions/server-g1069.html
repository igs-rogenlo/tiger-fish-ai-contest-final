Protocol
📄 Protocol 文件

Protocol 約 1.5 小時 走到定版基準

背景 / 挑戰

福星6有主遊戲收集、Step-Up、特殊上輪、JP、settlement、recovery 等資料。若欄位分散在 protocol 外層，client / server 很容易各自解讀。

定案口徑：
spin / bonus_spin 客製資料放 extra_info；next_fever 客製資料放 sg_map；共用欄位保留 result、result_wheels。

AI 做了什麼

  • 1
    產出 v1 / v2 / v2_rev1 / v2_rev2 多版文件。
  • 2
    整理座標、symbol ID、feature wheel command、Step-Up data shape。
  • 3
    回覆 client alignment：Buy Bonus scope、fake packet、可寫死欄位。
  • 4
    把每次修正落成 Excel，降低人工重新排版時間。
人工介入估算：約 30%。人工負責規格決策與 client 問題判斷；AI 負責整理、版本落檔與欄位一致性。
