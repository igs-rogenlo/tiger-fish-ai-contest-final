每個 Agent 一份的標準作業規則，啟動即載入。以下為「輸出格式」段（節錄）：
## ⚠️ 輸出格式（最高優先）
使用者在手機上也會看，所以：
1. 結構化資料優先用清單，不要用 markdown 表格（手機欄位會跑掉）
2. 架構圖/流程圖/sequence/state/ER → 一律用 Mermaid，
   過 6-gate hardlock（含 vision check）再交付
3. 大量表格/報表/訊息 PNG → Pillow 產圖（venv + 中文字體）
4. 程式碼 → code block；檔案路徑 → 用 `file_path:line_number` 方便跳轉
5. 變更回報 → 固定三段：改了哪些檔案 / 新增哪些資源 / 有沒有碰共用模組
＊另含 Session 啟動序列、自驗證協議、per-user Git 等段落（敏感段不公開）。

✅ 自驗證 Gate（SELF_CHECK_PROTOCOL）交付前 AI 自己負責

交付前 Agent 自己跑契約檢查，不靠下游才發現問題。四個核心原則（節錄）：
🔒 契約優先：每類交付物配 HARDLOCK（圖/規格/code…），
   定義「什麼算完成」的可機器驗證 gate；條目用「動作＋通過條件」兩段式寫

🔒 自驗證循環：完成交付 → 跑契約 gates → 全 PASS？
   ├─ YES → 留證 → 交付
   └─ NO  → 進失敗處理（修交付物）

🔒 失敗不改標準：gate FAIL 不接受降標準/override；永遠修交付物（圖/spec/code），
   不修 gate；改契約須留 audit trail

🔒 可觀察留證：每次交付附通過證據
