🎨 換皮與改製基礎

換皮不是只換圖，整套表演素材 都要接上

🎯 背景 / 挑戰

福星6從福星5基底改製，但外觀主題、表演素材與部分流程節奏都已不同。依 commit log，換皮範圍包含主畫面、免費遊戲、收集條、特殊輪與多段特效。

因此換皮不是單純替換圖片，而是要把素材用途、場景位置、動畫觸發與新玩法流程一起整理。

換皮項目：
角色、報獎、bar條、MG/FG frame、左右 frame、圖騰、+1spin、獅子、特殊輪符號、金幣飛行特效。

🤖 AI 做了什麼

  • 1
    依美術串接說明，把福星6素材接到主遊戲、免費遊戲與收集條場景。
  • 2
    替換 ActiveSymbol / Scatter / 圖騰 / +1spin 等玩法可見素材。
  • 3
    調整 MG frame、FG frame、symbol 間距、special reel layout 與遮罩層級。
  • 4
    串接獅子進退場、金幣 VFX trail、特殊輪 prefab，讓換皮後仍能配合新玩法演出。
結果：福星6從主畫面到免費遊戲都有完整視覺識別；素材、場景與表演節奏能接上新玩法，不只是沿用福星5外殼。

🎯 背景 / 挑戰

免費遊戲不是單一動畫，而是一串連續表演。人工提供編導規格書原檔後，AI 依照表演順序把進場、金幣、收集、獅子與結算接成同一個節奏。

🤖 AI 做了什麼

  • 1
    依編導規格建立免費遊戲一手的演出順序。
  • 2
    金幣飛行與分數變化同步，避免玩家看到數字跳錯。
  • 3
    收集與重置時機往後收，避免畫面還沒結束就被清空。
  • 4
    獅子進退場、宣告與倒數節奏對齊，讓表演看起來完整。
  • 5
    處理快停、重播與銷毀時機，降低表演中斷或閃跳風險。
結果：免費遊戲從進場到結算形成完整演出，畫面節奏與規格流程一致。

大項目 ②
規格面：盤面與分數
🪙 規格面 · 盤面與分數

把福星6的 分數規格 轉成玩家看得懂的畫面

🤖 AI 釐清的規格

  • 1
    金幣飛行後，盤面要呈現規格給定的結果，不用舊遊戲的累加方式。
  • 2
    新轉出的盤，也要能接住當手金幣效果，畫面不能漏加。
  • 3
    收集順序需符合福星6規格，避免先後順序造成分數錯位。
  • 4
    前端只負責把規格結果正確表現，不自行推測另一套結果。

🎯 呈現重點

  • A
    玩家視角：看到的是「金幣飛到哪裡、哪個盤被加分、收集後總分變化」。
  • B
    規格視角：每一步都要照福星6定義的結果顯示，不被福星5舊規則影響。
  • C
    表演視角：分數變化要跟飛行、收集動畫同步，避免畫面與結果脫節。
結果：盤面、金幣與收集結果能用一致的畫面語言呈現，玩家看到的分數就是規格要的結果。

📊 驗證面 · 完整流程

不只看單點，改用 整輪流程 驗證

🎯 背景 / 挑戰

測試資料不是只針對單一畫面或單一功能點，而是依照 server 制定的 protocol 文件建立，覆蓋本作從 maingame feature 進入 freegame，再到 freegame feature 的完整遊玩流程。

因此驗證重點不是「某一格數字有沒有對」，而是整條 protocol 流程中，狀態、表演、盤面與結果是否能一路連貫。

🤖 AI 做了什麼

  • 1
    依 server protocol 檢查測試資料格式與流程順序。
  • 2
    確認 maingame feature、freegame、freegame feature 三段銜接正確。
  • 3
    用完整流程資料反覆跑驗證，確認前端表演與盤面結果一致。
驗證重點：
server protocol → maingame feature → freegame → freegame feature → 前端表演與盤面顯示。
下一步： 串接真實 server 資料測試，確認正式資料流下的流程與表演仍一致。

表演目標：

🤖 AI 做了什麼

  • 2
    進度增加時搭配命中特效，強化每次收集的回饋。
  • 3
    集滿時切換升級表演，讓玩家知道已達成階段目標。
  • 4
    控制進度回退與下一輪銜接，避免畫面突然跳回起點。
結果：主遊戲新增清楚的收集成長感，規格結果能被玩家用表演理解。
