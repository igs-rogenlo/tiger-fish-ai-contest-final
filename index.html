Recap · Problem & Goal

競賽回顧：痛點與目標

🔥 欲解決的開發痛點
• 重複性高 — 每台新機台都要重做類似的規格、素材、數值設定
• 知識散落 — 競品分析、舊規格的市場與產品經驗、美術設計建議散落各處，難以系統化傳承複用
• 迭代效率 — 修改調整需要大量人工時間重複操作
🎯 我們的目標
• 選定主題：新一代福星高照（翻新既有機台）
• 使用既有 Code / 素材為基底，專注驗證 AI 製程
• 為 6 個職能各建一個 AI Agent，具備完整知識庫
• 建立可複製到其他機台的框架

Technical Highlights

技術實作亮點

🧠 記憶 + 學習迴路
記住歷次修正、跨 Agent 共享上下文、交付前自檢修正，越用越準的正向循環。
⚡ 自動生成 Skill
對話即更新 — Agent 自動萃取經驗成可執行能力模組，打破手動維護 Prompt／知識庫的舊模式。
🛠 AI 工具鏈
每個 Agent 配置專屬生成式／自動化／分析工具，以 MCP・CLI・API・網頁自動化操作。
🔒 權限 / 資安
獨立權限範圍、分級授權、機敏資料不外洩、操作日誌完整記錄。
📊 雙儀表板
職責審計儀表板（使用狀況／需求缺口）＋ 健康度監控儀表板（即時告警）。
🚀 可擴展設計
新增 Agent 只需灌知識 + 設工具鏈，不動核心；架構支援隨時擴充。
