Why Not Just a Personal Agent

和「自己電腦裝 AI Agent」差在哪

現在人人都能裝 Claude Code／Cursor。差別不在「誰的 Agent 比較聰明」，而在 —— 個人 Agent 讓「一個人」做得更快；我們的框架讓「一個組織」能複製、能累積、管得住。

Technical Highlights

技術實作亮點

🧠 記憶 + 學習迴路
記住歷次修正、交付前自檢，越用越準的正向循環（各 Agent 獨立記憶 scope）。
⚡ 自動生成 Skill
對話即更新，Agent 自動把經驗變成可執行能力模組，打破手動維護知識庫的舊模式。
🛠 AI 工具鏈
每個 Agent 配置專屬生成式／自動化工具，以 MCP・CLI・API・瀏覽器自動化操作。
🔒 多用戶 × 權限隔離
一個 Agent 同時服務多位使用者，各自獨立：分級授權（讀/寫/執行）、per-user Git 權限、資料與操作日誌逐人隔離。
🧠 跨機台知識複用
既有專案的框架、規格、Code 萃取入庫，下一台機台直接複用，不從零開始。
🚀 可擴展設計
新增 Agent 只需灌知識 + 設工具鏈，不動核心。
📄 每個 Agent 的 AGENTS.md ＋ docs/system/SELF_CHECK_PROTOCOL.md — 規範不是口號，是可機器驗證的契約（書審佐證）：
🔒 契約優先：每類產出（圖/規格/code）配 HARDLOCK，定義「什麼算完成」的 gate　🔒 自驗證循環：完成→跑 gates→全 PASS 才交付 🔒 失敗不改標準：gate FAIL 只修產出、不降標準　🔒 可觀察留證：交付附通過證據 —— 交付前 AI 自己負責，不靠下游才發現問題
Harness Engineering

每個 Agent 都是「工程化」的，不只是一段 prompt

每個職能 Agent 背後，是一整套結構化、可維護、持續演進的工程結構（harness）—— 規範檔、協議文件、技能模組、知識庫，不是一句 prompt。這是「Agent 工程」，也是別人複製不來的深度。

📋 AGENTS.md 規則檔
每個 Agent 約 300 行標準作業規則（.md），snippet 自動注入、持續維護。
📐 系統協議 docs/system
自驗證、圖表硬鎖、開工鮮度檢查、啟動序列… 可機器驗證的契約。
🧩 Skills 技能模組
把 SOP 包成可執行技能（美術 Agent 約 18 個：Cocos 驗證／美術 QC／Spine 匯出…）。
🔧 TOOLS.md 工具路由
預配專屬工具鏈（MCP／CLI／API／瀏覽器自動化），開箱即用、統一維護。
🧠 知識庫 + 記憶
各職能獨立 knowledge/ ＋ 學習記憶（對話即更新、越用越強）。
✅ 自驗證 Gate + 權限
6-gate hardlock + vision check 交付前自查；per-user Git 權限隔離。
整套都是結構化檔案、持續演進 —— 換人接手、複製到下一台，harness 直接承接帶走。
Co-Editing with Agents · 已支援 / 前瞻

與 Agent 即時共編（已建置、為未來預備）

已自架、可用的「與 Agent 共編」能力：員工自然語言交辦，Agent 做到一段給連結，點開即時看／接手共編（設計 Penpot・Open Design，程式 VS Code）。本次福星高照 6 職能尚未實際採用，是為日後專案複雜度提高、共編需求變大時預先備好的基礎建設。

🎨 Penpot — 向量美術共編
Agent 邊做設計、做到一段給連結，員工點開接手共編。左圖為平台上已有的 Agent 共編實例（活動頁／Golden Tide，非本次福星高照）。
🖥️ Open Design — 網頁／儀表板共編
Agent 設計網頁/儀表板，做到一段給連結，員工點開即時檢視、接手調整。
💻 VS Code IDE — 程式共編／看 diff
Client/Server/機率：Agent 做到一段給 IDE 連結，員工直接看 diff、接手共改程式碼。
💬 Open WebUI — 團隊對話入口
瀏覽器直接與各職能 Agent（openclaw）對話。
