Why Not Just a Personal Agent

和「自己電腦裝 AI Agent」差在哪

現在人人都能裝 Claude Code／Cursor。差別不在「誰的 Agent 聰明」，而是定位不同 —— 個人 Agent 讓「一個人」做得更快、把機敏資料留在本機；我們的框架放大的是「整個組織」—— 越做越快、知識留存、集中治理。不是取代，是各守其位。

Technical Highlights

技術實作亮點

🧠 記憶 + 學習迴路
記住歷次修正、交付前自檢，越用越準的正向循環（各 Agent 獨立記憶 scope）。
⚡ Skill 淬鍊上線
對話經驗自動沉澱進記憶庫，再淬鍊成可執行 Skill（經人工審核上線）—— 不靠手動維護知識庫。
🛠 AI 工具鏈
每個 Agent 配置專屬生成式／自動化工具，以 MCP・CLI・API・瀏覽器自動化操作。
🔒 多用戶 × 權限隔離
一個 Agent 同時服務多位使用者，各自獨立：分級授權（讀/寫/執行）、per-user Git 權限、資料與操作日誌逐人隔離。
🧠 相似類型自動複用
做新機台時，AI 自動比對、挑出相似規格/類型的既有框架與 Code 直接複用，不從零開始。
🚀 可擴展設計
新增 Agent 只需灌知識 + 設工具鏈，不動核心。
💰 成本低且固定（不隨用量增加） —— 整套工具＝固定月訂閱約 US$340（Codex $200・Claude／Opus 4.8 $100・Kiro 備援 $20・Gemini $20，多與個人／他組共用）＋ 本地自架模型（零 API 費）。非按 API 用量計費 → 多做一台機台，月成本幾乎不變。

Limitations & Mitigations

已知限制與對策

不誇大、誠實揭露 —— 這套框架現階段的限制，以及我們怎麼控管：

⚠️ 限制
✅ 對策
AI 生圖有隨機性，美術一致性／風格穩定度待提升
→
導入 ComfyUI ＋ 圖像特化模型提升可控性與一致性；交付前 Cocos 引擎內自檢＋人工把關
加速幅度因規格類型／複雜度而異（本次各職能已實測列出大致幅度）
→
持續在更多規格／複雜度上驗證，並持續進化 Agent 能力與知識沉澱（越用越強）
雲端模型有資料邊界考量
→
機敏資料（如禮品序號）一律走本地自架模型，不外送雲端
誠實面對限制、持續進化 —— 這正是框架會越來越強的原因。
